  Calcul de la taille totale (content-box par défaut) :
  ┌─────────────────────────────────────────────────────┐

🟣 Responsive Design - Mobile First

Adapter la mise en page à tous les écrans avec des media queries

Breakpoints et stratégie Mobile-FirstResponsive
  MOBILE FIRST : on part du plus petit et on agrandit
  ─────────────────────────────────────────────────────

  ←─────── 320px ──────→  ←── 600px ──→  ←── 900px ──→  ←── 1200px ──→
  █ MOBILE                █ TABLETTE      █ LAPTOP        █ DESKTOP

  ┌─────────────┐     ┌────────────────┐  ┌──────────────────────────┐
  │   │LOGO│     │     │LOGO      NAV│  │LOGO  Accueil  Section  CTA│
  │  ┌─────────┐│     ├────────────────┤  ├──────────────────────────┤
  │  │  H E R O││     │    HERO          │  │     HERO   │  VISUEL     │
  │  │  (100%) ││     │   (100%)         │  │  (texte)  │  (card)     │
  │  └─────────┘│     ├────────────────┤  ├──────────────────────────┤
  │  ┌─────────┐│     │ ■ feat │ ■ feat│  │ ■ feature │ ■ feature│
  │  │ feature ││     │                │  │ ■ feature├──────────────│
  │  ├─────────┤│     │ ■ feat │ ■ feat│  ├──────────────────────────┤
  │  │ feature ││     ├────────────────┤  │ ■ card │ ■ card │ ■ card│
  │  └─────────┘│     │■ card │ ■ card│  └──────────────────────────┘
  └─────────────┘     └────────────────┘

  CODE CSS (mobile-first avec min-width) :
  ─────────────────────────────────────────
  /* Mobile : base */
  .features-grid { grid-template-columns: 1fr; }

  /* Tablette ≥ 600px */
  @media (min-width: 600px) {
    .features-grid { grid-template-columns: repeat(2, 1fr); }
  }

  /* Desktop ≥ 900px */
  @media (min-width: 900px) {
    .features-grid { grid-template-columns: repeat(3, 1fr); }
  }

🟡 Cycle de vie d'une requête fetch()

Comment JavaScript communique avec le serveur sans rechargement

fetch() async/await - Flux completJS
  NAVIGATEUR                         SERVEUR (ou fichier JSON)
  ─────────────────────────────────────────────────────────────

   1. await fetch('data/formations.json')
   │
   ├──────────── HTTP GET ───────────────────────────────────►
   │                                                          │
   │                                        200 OK + JSON data
   ◄──────────── Réponse ◄──────────────────────────────────┘
   │
   2. if (!response.ok) throw new Error()  ← Vérifier le status
   │
   3. const data = await response.json()   ← Parser le JSON
   │
   ├── Succès ──► afficherFormations(data.formations)
   │
   └── Erreur ──► catch(err) → afficherMessageErreur()
                 └── finally → masquerLoader()

  ─────────────────────────────────────────────────────────────
  ÉTATS POSSIBLES DE LA REQUÊTE :
  ┌────────────────┬─────────────────────────────────────────┐
  │ État           │ Action recommandée                      │
  ├────────────────┼─────────────────────────────────────────┤
  │ En attente     │ Afficher un spinner/loader              │
  │ 200 OK         │ Parser JSON et afficher les données     │
  │ 404 Not Found  │ Afficher "ressource introuvable"        │
  │ 500 Server Err │ Afficher "erreur serveur, réessayez"   │
  │ Réseau coupé   │ catch() → "vérifiez votre connexion"   │
  └────────────────┴─────────────────────────────────────────┘

  ⚠ fetch() ne rejette pas pour les erreurs HTTP (404, 500)
     Il faut vérifier manuellement : if (!response.ok) throw

🟡 La propagation des événements DOM

Comment les événements se propagent dans l'arbre DOM

Bubbling (propagation) et délégation d'événementsJS
  CAPTURE (haut → bas)          BUBBLING (bas → haut)
  ─────────────────────          ───────────────────────

   document                       document
      │ ▼                              ▲ │
   <html>                           <html>
      │ ▼                              ▲ │
   <body>                           <body>
      │ ▼                              ▲ │
   <div>                            <div>
      │ ▼                              ▲ │
   <button> ◄── 👆 CLIC ICI       <button>

  La phase de BUBBLING (remontée) permet la DÉLÉGATION :
  ┌─────────────────────────────────────────────────────┐
  │  ✗ MAUVAIS (1 listener par bouton)                 │
  │  boutons.forEach(btn => btn.addEventListener(...))  │
  │                                                     │
  │  ✓ BON (délégation sur le parent)                  │
  │  parent.addEventListener('click', (e) => {          │
  │    if (e.target.matches('.btn-filtre')) { ... }      │
  │  })                                                 │
  │                                                     │
  │  Avantage : fonctionne même pour les éléments       │
  │  ajoutés dynamiquement après coup                   │
  └─────────────────────────────────────────────────────┘

  event.stopPropagation() ── arrête la remontée
  event.preventDefault()  ── annule le comportement natif
                              (ex: submit → ne recharge pas)

🟢 Accessibilité - WCAG & ARIA

Rendre les interfaces utilisables par tous

Les 4 principes WCAG + ARIA en pratiqueAccessibilité
  LES 4 PRINCIPES WCAG (POUR) :
  ┌─────────────┬──────────────────────────────────────────┐
  │ Perceptible  │ alt sur images, sous-titres vidéos,       │
  │              │ contraste suffisant (≥ 4.5:1)              │
  ├─────────────┼──────────────────────────────────────────┤
  │ Utilisable   │ navigation clavier (Tab/Enter/Espace),     │
  │              │ focus visible, pas de piège clavier        │
  ├─────────────┼──────────────────────────────────────────┤
  │ Robuste      │ HTML valide W3C, ARIA correct,             │
  │              │ compatible lecteurs d'écran                │
  ├─────────────┼──────────────────────────────────────────┤
  │ Omnipréhensible│ langue déclarée, labels clairs,          │
  │              │ messages d'erreur explicites               │
  └─────────────┴──────────────────────────────────────────┘

  ARIA - QUAND ET COMMENT :
  ┌───────────────────────────────────────────────────────┐
  │                                                       │
  │  ✓ Menu burger accessible :                          │
  │  <button aria-expanded="false"                       │
  │          aria-controls="mobile-menu"                 │
  │          aria-label="Ouvrir le menu">                │
  │                                                       │
  │  ✓ Contenu dynamique (formations chargées) :         │
  │  <div aria-live="polite">                            │
  │  → annonce les mises à jour aux lecteurs d'écran      │
  │                                                       │
  │  ✓ Navigation entre sections :                      │
  │  <nav aria-label="Navigation principale">            │
  │  <section aria-labelledby="titre-section">          │
  │                                                       │
  │  ✗ Ne pas abuser de ARIA :                          │
  │  "No ARIA is better than bad ARIA"                    │
  │  Toujours préférer une balise HTML sémantique native  │
  └───────────────────────────────────────────────────────┘

  CONTRASTE COULEUR (WCAG AA) :
  ┌──────────────────┬──────────────────────────────────┐
  │ Texte normal     │ ratio ≥ 4.5:1                  │
  │ Texte large      │ ratio ≥ 3:1                    │
  │ Composants UI    │ ratio ≥ 3:1                    │
  └──────────────────┴──────────────────────────────────┘
  Outil : WebAIM Contrast Checker
